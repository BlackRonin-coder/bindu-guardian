# Public Release Boundary

## Purpose

This repository is designed to show what Bindu Guardian is, what problem it addresses, and how it is framed architecturally.

## Publicly visible

- architecture
- positioning
- examples
- safe demo shell
- project context

## Intentionally withheld

- protected implementation details
- internal scoring models
- optimisation logic
- operational thresholds
- sensitive deployment methods

## Why this boundary exists

Bindu Guardian is intended for high-risk systems. Public communication must preserve understanding without unnecessarily exposing control logic or protected design leverage.
